Sección 1 · Introducción

Programación Orientada a Objetos con C++

Hasta ahora programabas dando órdenes paso a paso. La Programación Orientada a Objetos (POO) cambia la forma de pensar: en vez de solo instrucciones sueltas, modelas el programa como un conjunto de objetos que tienen datos y saben hacer cosas — igual que en la vida real.

La idea en una frase

Un objeto junta en un solo lugar qué sabe (sus datos) y qué hace (sus acciones). Pensar en objetos hace que los programas grandes sean más fáciles de entender, corregir y reutilizar.

Nuestro proyecto integrador: un Cajero Automático 🏧

Todo este cuaderno gira en torno a un ejemplo que conoces de sobra: el cajero automático. Es perfecto para aprender POO porque un cajero es, naturalmente, un objeto:

En el cajero real…En POO se llama…
El saldo, el número de cuenta, el NIPAtributos (los datos que el objeto guarda)
Depositar, retirar, consultar saldoMétodos (las acciones que el objeto sabe hacer)
Una cuenta bancaria concreta (la tuya)Un objeto (un ejemplar real de la clase)
El "molde" de cómo es cualquier cuentaUna clase (la plantilla para crear objetos)

¿Por qué repasamos C++ antes de la Unidad 2?

La Unidad 2 (herencia, polimorfismo, patrones de diseño) se construye encima de lo que viste en el R.A. 1.2. Si las bases están firmes, lo nuevo entra fácil. Por eso este cuaderno repasa los cuatro pilares que necesitas dominar:

  • Entrada y salida: cómo el programa lee datos del usuario y muestra resultados (cin, cout).
  • Sentencias de control: cómo el programa decide (if, switch) y repite (for, while).
  • Arreglos y estructuras: cómo guardar muchos datos juntos.
  • Clases y objetos: el corazón de la POO — definir clases con atributos y métodos, y crear objetos con constructores.

Sección 2 · Repaso de C++

Entrada y salida de datos

Todo programa útil necesita comunicarse: mostrar información al usuario y recibir datos de él. En C++ eso se hace con cout (salida) y cin (entrada).

El esqueleto de todo programa C++

#include <iostream>   // librería de entrada/salida
using namespace std;  // para escribir cout en vez de std::cout

int main() {
    cout << "¡Bienvenido al cajero!" << endl;
    return 0;
}
Parte¿Qué hace?
#include <iostream>Trae las herramientas de entrada/salida (cin, cout).
int main()El punto donde arranca el programa. Todo lo de adentro se ejecuta en orden.
return 0;Le avisa al sistema que el programa terminó bien.

Salida: cout

El operador << "empuja" lo que quieras mostrar hacia la pantalla. endl salta de renglón.

double saldo = 1500.50;
cout << "Tu saldo es: $" << saldo << endl;
cout << "Gracias por su preferencia" << endl;

Puedes encadenar texto y variables en la misma línea con varios <<. La salida sería: Tu saldo es: $1500.5

Entrada: cin

El operador >> toma lo que el usuario teclea y lo guarda en una variable.

double monto;
cout << "¿Cuánto desea retirar? ";
cin >> monto;                       // el usuario teclea, p.ej., 500
cout << "Retirando $" << monto << endl;

Tipos de datos que más usarás

TipoGuardaEjemplo en el cajero
intNúmeros enterosint nip = 1234;
doubleNúmeros con decimalesdouble saldo = 1500.50;
charUn solo carácterchar opcion = 'R';
stringTexto (requiere #include <string>)string titular = "Ana López";
boolVerdadero/falsobool autorizado = true;
Sobre el formato de decimales

Por defecto C++ muestra 1500.5, no 1500.50. Para dinero, puedes fijar 2 decimales con #include <iomanip> y cout << fixed << setprecision(2); antes de imprimir. Lo verás en el ejemplo guiado.

Sección 3 · Repaso de C++

Sentencias de control

Un cajero no hace siempre lo mismo: decide (¿hay saldo suficiente?) y repite (mostrar el menú hasta que el usuario salga). Para eso están las sentencias de control.

A · Selección: tomar decisiones

if / else — el camino que se bifurca

double saldo = 1500;
double retiro = 2000;

if (retiro <= saldo) {
    saldo = saldo - retiro;
    cout << "Retiro exitoso. Nuevo saldo: " << saldo << endl;
} else {
    cout << "Fondos insuficientes." << endl;
}

Si la condición entre paréntesis es verdadera, se ejecuta el primer bloque; si no, el del else. Los operadores de comparación: == (igual), != (distinto), >, <, >=, <=.

switch — elegir entre muchas opciones

Ideal para el menú del cajero: según la opción que teclee el usuario, hace una cosa u otra.

int opcion;
cout << "1) Consultar  2) Depositar  3) Retirar  4) Salir\n";
cin >> opcion;

switch (opcion) {
    case 1:
        cout << "Mostrando saldo..." << endl;
        break;
    case 2:
        cout << "Depositando..." << endl;
        break;
    case 3:
        cout << "Retirando..." << endl;
        break;
    default:
        cout << "Opción no válida" << endl;
}
⚠ No olvides el break

Sin break al final de cada case, el programa "se sigue de largo" y ejecuta también los casos siguientes. El default es lo que pasa si ninguna opción coincide.

B · Repetición: hacer algo varias veces

for — cuando sabes cuántas veces

// Mostrar los últimos 3 movimientos
for (int i = 1; i <= 3; i++) {
    cout << "Movimiento " << i << endl;
}

El for tiene tres partes: inicio (int i = 1), condición para seguir (i <= 3) y avance (i++, suma 1 cada vuelta).

while — mientras se cumpla una condición

int intentos = 0;
int nip;
while (intentos < 3) {
    cout << "Ingrese su NIP: ";
    cin >> nip;
    if (nip == 1234) break;   // NIP correcto, salimos
    intentos++;
}

do while — al menos una vez

Perfecto para el menú: siempre se muestra al menos una vez, y se repite hasta que el usuario elija salir.

int opcion;
do {
    cout << "1) Consultar  2) Salir\n";
    cin >> opcion;
    // ... procesar opción ...
} while (opcion != 2);    // repite hasta que elija "Salir"
¿for, while o do-while?

for: sabes el número de repeticiones (recorrer 3 movimientos). while: repites mientras algo sea cierto, y podría no ejecutarse ni una vez. do-while: igual que while, pero se ejecuta mínimo una vez (ideal para menús).

Sección 4 · Repaso de C++

Arreglos y estructuras

Un cajero maneja muchos datos: el historial de movimientos, los billetes disponibles, los datos de varias cuentas. Para guardar colecciones de datos usamos arreglos y estructuras.

Arreglos unidimensionales — una lista

Un arreglo guarda varios valores del mismo tipo bajo un solo nombre. Cada valor tiene una posición (índice), empezando en 0.

// Los últimos 3 retiros
double retiros[3] = {500.0, 1000.0, 200.0};

cout << retiros[0];   // 500.0  (primer elemento, índice 0)
cout << retiros[2];   // 200.0  (tercer elemento, índice 2)

retiros[1] = 750.0;   // cambiar el segundo valor

Con un for recorres todo el arreglo:

double total = 0;
for (int i = 0; i < 3; i++) {
    total = total + retiros[i];
}
cout << "Total retirado: " << total;   // 1450

Arreglos multidimensionales — una tabla

Un arreglo de dos dimensiones es como una tabla con filas y columnas. Por ejemplo, los saldos de 2 cuentas en 3 meses:

double saldos[2][3] = {
    {1000, 1200, 1500},   // cuenta 0
    {500,  800,  600}     // cuenta 1
};
cout << saldos[0][2];   // 1500 (cuenta 0, mes 2)
cout << saldos[1][0];   // 500  (cuenta 1, mes 0)

Estructuras (struct) — agrupar datos distintos

Un arreglo guarda muchos datos del mismo tipo. Una estructura agrupa datos de distintos tipos que describen una misma cosa. Por ejemplo, una cuenta tiene número (int), titular (string) y saldo (double):

struct Cuenta {
    int numero;
    string titular;
    double saldo;
};

// Crear y usar una estructura:
Cuenta c1;
c1.numero = 1001;
c1.titular = "Ana López";
c1.saldo = 1500.50;

cout << c1.titular << " tiene $" << c1.saldo;

El punto (.) accede a cada campo de la estructura.

struct: el escalón previo a las clases

Una struct ya junta datos relacionados — es el primer paso hacia la POO. La gran diferencia: en la siguiente sección verás que una clase agrupa los datos Y TAMBIÉN las acciones (métodos), y además protege esos datos. La clase es la struct llevada al siguiente nivel.

Sección 5 · El corazón de la POO

Clases y objetos

Llegamos a lo esencial. Una clase es el molde; un objeto es lo que sale del molde. La clase Cuenta describe cómo es cualquier cuenta; cada cuenta concreta (la de Ana, la de Luis) es un objeto.

Anatomía de una clase

Una clase tiene dos cosas: atributos (los datos que guarda) y métodos (las acciones que sabe hacer).

class Cuenta {
  private:
    double saldo;        // ATRIBUTO: dato que guarda

  public:
    void depositar(double monto) {    // MÉTODO: acción
        saldo = saldo + monto;
    }
    void mostrarSaldo() {             // MÉTODO: acción
        cout << "Saldo: $" << saldo << endl;
    }
};
Elemento¿Qué es?
class Cuenta { ... };Define el molde. Ojo: lleva punto y coma al final.
saldoAtributo: el dato que cada cuenta guarda.
depositar(), mostrarSaldo()Métodos: lo que la cuenta sabe hacer.

Crear y usar objetos

Una vez definida la clase, creas objetos a partir de ella y los usas con el punto (.):

int main() {
    Cuenta cuentaDeAna;          // crear un objeto de la clase Cuenta
    cuentaDeAna.depositar(1500); // usar un método del objeto
    cuentaDeAna.mostrarSaldo();  // Saldo: $1500
    return 0;
}

Control de acceso: public, private, protected

Aquí está la gran ventaja de las clases sobre las estructuras: pueden proteger sus datos. Esto se llama encapsulamiento.

Modificador¿Quién puede acceder?Uso típico
privateSolo la propia clase, desde adentroLos datos sensibles (saldo, NIP)
publicCualquiera, desde afueraLos métodos que el usuario usa (depositar)
protectedLa clase y sus "hijas" (herencia)Se verá a fondo en la Unidad 2
🔒 ¿Por qué hacer saldo privado?

Si el saldo fuera public, cualquiera podría escribir cuentaDeAna.saldo = 999999; y robar dinero sin control. Al hacerlo private, la única forma de cambiarlo es a través de métodos como depositar() y retirar(), que pueden validar (¿hay fondos?, ¿el monto es positivo?). Eso es proteger la integridad de los datos — justo lo que un cajero real debe hacer.

Sección 6 · El corazón de la POO

Constructor y destructor

Cuando abres una cuenta, no empieza "en blanco": tiene un titular y un saldo inicial. El constructor es el método especial que da esos valores iniciales al nacer el objeto. El destructor es lo que ocurre cuando el objeto deja de existir.

El constructor — dar vida al objeto

El constructor es un método que se llama igual que la clase y no devuelve nada (ni siquiera void). Se ejecuta automáticamente al crear el objeto.

class Cuenta {
  private:
    string titular;
    double saldo;

  public:
    // CONSTRUCTOR: mismo nombre que la clase
    Cuenta(string nombre, double saldoInicial) {
        titular = nombre;
        saldo = saldoInicial;
        cout << "Cuenta creada para " << titular << endl;
    }

    void mostrarSaldo() {
        cout << titular << ": $" << saldo << endl;
    }
};

Ahora, al crear el objeto, le pasas los datos iniciales entre paréntesis:

int main() {
    Cuenta cuentaDeAna("Ana López", 1500.0);  // el constructor corre aquí
    cuentaDeAna.mostrarSaldo();                // Ana López: $1500
    return 0;
}

El destructor — la despedida del objeto

El destructor se llama igual que la clase pero con una virgulilla ~ adelante. Se ejecuta automáticamente cuando el objeto deja de existir (al terminar el programa o salir de su ámbito). Sirve para "limpiar" antes de irse.

class Cuenta {
    // ... atributos y constructor ...
  public:
    ~Cuenta() {     // DESTRUCTOR
        cout << "Cerrando la cuenta de " << titular << endl;
    }
};
Constructor y destructor: el ciclo de vida

Piénsalo como abrir y cerrar una cuenta en el banco: el constructor es el papeleo de apertura (le das nombre y depósito inicial), y el destructor es el cierre (se ejecuta solo, al final). Tú no los llamas manualmente: C++ los dispara automáticamente al crear y destruir el objeto.

Referencias y asignación de objetos

Puedes tener varios objetos de la misma clase, cada uno con sus propios datos:

Cuenta cuentaAna("Ana López", 1500.0);
Cuenta cuentaLuis("Luis Pérez", 800.0);
// cuentaAna y cuentaLuis son objetos INDEPENDIENTES:
// cada uno guarda su propio titular y su propio saldo.
Sección 7 · Ejemplo guiado

El Cajero completo, paso a paso

Vamos a juntar TODO lo repasado en un solo programa: una clase Cuenta con atributos privados, constructor, métodos con validación, y un main que usa un menú con switch y un ciclo do-while. Léelo despacio: cada pieza ya la conoces.

El programa completo

#include <iostream>
#include <iomanip>   // para setprecision (formato de dinero)
using namespace std;

class Cuenta {
  private:
    string titular;
    double saldo;

  public:
    // Constructor: da valores iniciales
    Cuenta(string nombre, double saldoInicial) {
        titular = nombre;
        saldo = saldoInicial;
    }

    // Método con validación: solo deposita montos positivos
    void depositar(double monto) {
        if (monto > 0) {
            saldo = saldo + monto;
            cout << "Depósito exitoso." << endl;
        } else {
            cout << "Monto inválido." << endl;
        }
    }

    // Método con validación: no permite sobregiro
    void retirar(double monto) {
        if (monto > saldo) {
            cout << "Fondos insuficientes." << endl;
        } else if (monto <= 0) {
            cout << "Monto inválido." << endl;
        } else {
            saldo = saldo - monto;
            cout << "Retiro exitoso." << endl;
        }
    }

    void consultar() {
        cout << fixed << setprecision(2);
        cout << titular << ", su saldo es: $" << saldo << endl;
    }
};

int main() {
    Cuenta miCuenta("Ana López", 1500.0);
    int opcion;
    double monto;

    do {
        cout << "\n--- CAJERO ---" << endl;
        cout << "1) Consultar saldo" << endl;
        cout << "2) Depositar" << endl;
        cout << "3) Retirar" << endl;
        cout << "4) Salir" << endl;
        cout << "Opción: ";
        cin >> opcion;

        switch (opcion) {
            case 1:
                miCuenta.consultar();
                break;
            case 2:
                cout << "Monto a depositar: ";
                cin >> monto;
                miCuenta.depositar(monto);
                break;
            case 3:
                cout << "Monto a retirar: ";
                cin >> monto;
                miCuenta.retirar(monto);
                break;
            case 4:
                cout << "Gracias por su preferencia." << endl;
                break;
            default:
                cout << "Opción no válida." << endl;
        }
    } while (opcion != 4);

    return 0;
}
